### M4: Save/Export
- [x] **`.mneme-map` export** — `onExportMnemeMap()` builds `MnemeMapExport` JSON with stars, tables, parameters, and MWG systems
- [x] **`.mneme-map` import** — `onImportMnemeMap()` restores full sector state, switches to Generate mode, updates UI controls
- [x] **`mapName` variable** — used in export filename and import restoration
- [x] **`MnemeMapExport` type** — now actively used in export/import pipeline

### QA-006 — Recursive Sextet Protocol (RSP) / Nested Sextet Addressing (NSA)
- [x] **Replaced `d66` with `d6666` for spherical coordinates** — 1,296 outcomes per axis instead of 36
  - XY azimuth resolution: 10° → ~0.278°
  - Z elevation resolution: ~5.14° → ~0.139°
  - Visual clustering eliminated; stars scatter uniformly across the sphere
- [x] **`src/dice.ts`** — added `rollD6666()`, `d6666ToIndex()`, `d6666ToDegrees()`, `d6666ToElevation()`
- [x] **`src/generator.ts`** — uses `rollD6666()` for `xyRoll` and `zRoll`; stores tuples `[number, number]` on `GeneratedStar`
- [x] **`src/types.ts`** — `xyRoll` and `zRoll` typed as `[number, number]`
- [x] **`RECURSIVE_SEXTET_PROTOCOL.md`** — full protocol documentation

### Files changed
- `src/main.ts` — `renderTables()` calls, `.mneme-map` save/load, MWG JSON attach
- `src/ui.ts` — export/import buttons, context panel "Load MWG JSON", new callback params
- `src/types.ts` — roll tuples for d6666
- `src/generator.ts` — d6666 coordinate generation, roll tuple storage
